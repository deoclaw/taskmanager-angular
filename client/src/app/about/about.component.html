About This App

Project Report

This is a task management app built using the MEAN stack (MongoDB, Express, Angular, and Node.js) for a class project

This project report contains an overview of the app and its features, challenges faced, and future improvements.

Overview

Features include:

  • Basic CRUD operations to a cloud-based MongoDB collection
    • Create a new task
    • Read all tasks
    • Updated (edit) a task
    • Delete (complete) a task
  • RESTful API
  • Routing between pages
  • Integration of Angular Material CDK
  • An About page

The front end code can be found this repository and the back end code can be found at this repository

Challenges

There were a number of challenges with this app. The backend went pretty smoothly; however, Angular was so different than previous front-end frameworks I had been introduced to and the documentation was not very user friendly. I found videos made by MongoDB of all places that went over Angular concepts, which helped.

One of the biggest challenges was security. Currently, the server only runs on the local machine rather than being accessible publicly. This is because I was unable to adequately integrate authentication and authorization. While I found a tutorial on JWT integration for MERN-stack apps by David Gray who taught Backend Development I, I found myself unsure of how to integrate it well in Angular. Rather than potentially introduce a huge security hole, I have opted to keep this a local deployment instead.

Another major challenge was using Angular Material. Angular's documentation does not readily give users information about the different variables for color used. Eventually I found the guide to system variables which had a link to the Material color system which had, nestled in it, the guide to all the color variable names; however, for purpoting user friendly design the documentation is anything but.

Future Improvements

Future improvements to this project would first be to integrate a user system so users can link their tasks to their username. This would also necessitate a user authentication and authorization system so that this can be served publicly on a platform like render.

Another improvement would be getting the form validator to work and allow forward slashes for dates. Currently, it allows only some regex but not all (the goal being to keep anyone from submitting untrusted code.)
